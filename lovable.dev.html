Lovable.dev

Lovable is a full-stack AI development platform for building, iterating, and deploying web applications from natural language prompts.

  • Lovable helps create web apps and websites by describing the desired product in plain language.
  • It can generate frontend, backend, database, authentication, and integrations depending on the project needs.
  • The output is real editable code, so developers can inspect, change, and connect it to normal engineering workflows.
  • It is useful for prototypes, internal tools, MVPs, landing pages, and fast product experiments.

  • Describe the app goal, target users, main pages, data model, and expected actions.
  • Mention the visual style only after the core workflow is clear.
  • Attach screenshots or references when you want Lovable to match an existing interface.

  • Ask for one feature or fix at a time instead of rewriting the whole app repeatedly.
  • Review the preview after each change and name exactly what should be different.
  • Keep accepted behavior stable by saying what should not change.

  • Use GitHub integration when you want source control, manual review, or team collaboration.
  • Test authentication, database rules, payments, forms, and edge cases before treating the app as production-ready.
  • Keep secrets, API keys, and permissions out of generated client-side code.

  • Good first prompt: "Build a task management app for a small team with login, projects, task status, due dates, and a dashboard."
  • Good edit prompt: "On the task detail page, add comments with author name and timestamp. Do not change the dashboard layout."
  • Good debug prompt: "The save button shows success, but refreshed data is missing. Inspect the database write path and fix the root cause."
  • Good design prompt: "Make the table denser for daily use, keep the current colors, and add clear empty and loading states."

  • Fast path from idea to working app preview.
  • Helpful for non-boilerplate UI generation and quick iteration.
  • Supports a conversational workflow for adding features and fixing issues.
  • GitHub integration can bring generated code into a normal repository workflow.

  • Generated code still needs developer review for security, performance, accessibility, and maintainability.
  • AI may make assumptions about business rules unless the prompt is specific.
  • Production apps need real testing around auth, data access, payments, and failure states.
  • Keep a habit of committing useful milestones so you can recover from unwanted changes.

Lovable itself runs as a cloud platform, but the generated project code can be worked on locally after connecting the project to GitHub.

  • Connect the Lovable project to GitHub.
  • Clone the GitHub repository to your machine.
  • Install dependencies and run the app locally based on the generated project's README or package scripts.
  • Push commits back to GitHub so changes can sync with Lovable.

Lovable can connect to Supabase to use PostgreSQL database, authentication, storage, real-time updates, and backend edge functions. You can also use Lovable Cloud, which provides a built-in backend based on Supabase without separate setup.

  1. Create or sign in to your Supabase account.
  2. Create a Supabase project, or let Lovable guide you while connecting.
  3. In Lovable, open the project editor and go to Integrations or project settings.
  4. Select Supabase, click Connect Supabase, sign in, and authorize Lovable.
  5. Choose the Supabase organization and project to link with your Lovable app.
  6. Wait until Lovable confirms that Supabase is connected.

To connect app screens with database tables, describe the data feature in Lovable chat. For example: "Create a feedback form and save name, email, rating, and message in Supabase." Lovable can suggest the table structure and SQL. Run the SQL in Supabase SQL Editor, then confirm in Lovable so it can wire the UI to the database.

  • Use Supabase Table Editor to view and edit rows.
  • Use SQL Editor for table creation, schema updates, and custom queries.
  • Use Authentication for users and login flows.
  • Use Storage for files such as images and documents.
  • Use Edge Functions when the app needs secure backend logic or API secrets.

If your Lovable project is connected to Supabase, you can see backend tables from the Supabase dashboard.

  1. Open your Supabase dashboard.
  2. Select the project connected with Lovable.
  3. Go to Table Editor from the left menu.
  4. Select a table to view columns, rows, and relationships.
  5. Use SQL Editor if you want to inspect tables with SQL queries.

In Lovable, you can also ask: "Show me the database schema" or "Which Supabase tables does this app use?" to understand how the app is connected to the backend.

In Lovable, creating a copy of a project is called remixing. A remix creates a separate copy in the currently selected workspace, while the original project stays unchanged.

  1. Open the Lovable project you want to copy.
  2. Click the project name at the top left of the editor.
  3. Open Project Settings.
  4. Select Remix.
  5. Choose the workspace where the copied project should be created.
  • You can remix your own projects or projects you have access to.
  • Other users can remix your project only if remixing/access is enabled for them.
  • If the project uses Lovable Cloud backend, backend structure like tables and edge functions can be copied, but existing data and manually added secrets are not copied.
  • Projects connected to Supabase may have remix limitations, especially when remixing projects owned by other users.
