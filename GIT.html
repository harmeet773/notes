Git identity is the name and email attached to your commits. This is different from authentication. Identity controls how your commits appear in Git history and on GitHub.

1. Set identity for all repositories on your machine

Use global config when you want the same identity everywhere by default. 

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

2. Set identity only for the current repository

Use local config when one repository should use a different name or email. 

git config user.name "Project Specific Name"
git config user.email "project@example.com"

3. Check which identity is active

git config user.name
git config user.email
git config --list --show-origin

--show-origin helps you see whether the value comes from the system, global, or local configuration file.

4. Common ways to manage identity

  • Global only: best when you use one name and email for all repositories.
  • Global + local override: best when most repositories use one identity, but a few need a different one.
  • Conditional config with includeIf: useful when work and personal repositories are stored in different folders.

5. Example of conditional identity setup

This approach automatically applies a different identity based on the folder where the repository is located. 

git config --global includeIf.gitdir:~/work/.path ~/.gitconfig-work
git config --global includeIf.gitdir:~/personal/.path ~/.gitconfig-personal

Example config files:

[user]
  name = Your Work Name
  email = work@example.com
[user]
  name = Your Personal Name
  email = personal@example.com

6. Good practice

Set a global default first, then override it locally for repositories that need a different identity. If you regularly switch between work and personal repositories, conditional config is cleaner.

Use separate SSH keys and a dedicated SSH config so Git can decide which GitHub account to use for each repository.

1. Create 2 SSH keys

ssh-keygen -t ed25519 -C "personal@example.com"
ssh-keygen -t ed25519 -C "work@example.com"

Save them with different names such as id_ed25519_personal and id_ed25519_work.

2. Add the keys to the SSH agent

ssh-add ~/.ssh/id_ed25519_personal
ssh-add ~/.ssh/id_ed25519_work

3. Add both public keys to GitHub

Copy each .pub file and add it to the correct GitHub account from Settings > SSH and GPG keys.

4. Configure SSH aliases

Create or update ~/.ssh/config:

Host github-personal
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_personal

Host github-work
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_work

5. Clone repositories with the correct alias

git clone git@github-personal:your-personal-username/repo-name.git
git clone git@github-work:your-work-username/repo-name.git

6. Set the right Git identity per repository

git config user.name "Your Personal Name"
git config user.email "personal@example.com"
git config user.name "Your Work Name"
git config user.email "work@example.com"

7. Test both accounts

ssh -T git@github-personal
ssh -T git@github-work

Each command should confirm authentication for the matching GitHub account.
